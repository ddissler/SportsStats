# SportsStats ‚Äî Setup & Publishing Guide

## Prerequisites

1. **.NET 8 SDK** ‚Äî already installed (used to build this project)
2. **Node.js** ‚Äî already installed
3. **SQL Server** ‚Äî default instance on `localhost` ‚úì (auto-detected)
4. **ASP.NET Core Hosting Bundle for .NET 8** ‚Äî required for IIS
   - Download from: https://dotnet.microsoft.com/en-us/download/dotnet/8.0
   - Choose "Hosting Bundle" under the Runtime column
   - This installs the ASP.NET Core Module (ANCM) into IIS

---

## Step 1 ‚Äî Get your API-Sports key

1. Go to https://api-sports.io and create a free account
2. Copy your API key from the dashboard
3. Open `SportsStats.API\appsettings.json`
4. Replace `YOUR_API_SPORTS_KEY` with your actual key:

```json
"ApiSports": {
  "ApiKey": "paste-your-key-here"
}
```

---

## Step 2 ‚Äî Run in Development (recommended before IIS)

Open **two terminals** from the repo root:

**Terminal 1 ‚Äî .NET API**
```
cd SportsStats.API
dotnet watch run
```
API starts at `https://localhost:7135`
Swagger UI: `https://localhost:7135/swagger`

**Terminal 2 ‚Äî React frontend**
```
cd client
npm run dev
```
App opens at `http://localhost:5173`
All `/api/*` calls proxy to the .NET server automatically.

## Step 3 ‚Äî Publish to IIS

### 3a. Build React and copy to wwwroot
```
cd client
npm run build
xcopy /E /Y dist\* ..\SportsStats.API\wwwroot\
```

### 3b. Publish .NET app
```
cd SportsStats.API
dotnet publish -c Release -o C:\inetpub\SportsStats
```

### 3c. Configure IIS

1. Open **IIS Manager** (`Win + R` ‚Üí `inetmgr`)
2. In the left tree, right-click **Sites** ‚Üí **Add Website**
3. Fill in:
   - **Site name**: `SportsStats`
   - **Physical path**: `C:\inetpub\SportsStats`
   - **Binding type**: `http`
   - **IP address**: `127.0.0.1`
   - **Port**: `80` (or `8080` if port 80 is taken)
   - **Host name**: *(leave blank)*
4. Click **OK**

### 3d. Set App Pool to No Managed Code

1. In IIS Manager, click **Application Pools** in the left tree
2. Find `SportsStats`, right-click ‚Üí **Basic Settings**
3. Set `.NET CLR version` to **No Managed Code**
4. Click OK

### 3e. Grant SQL Server access to IIS App Pool

Option A (easiest): Change the App Pool identity to **NetworkService**
1. Right-click `SportsStats` App Pool ‚Üí **Advanced Settings**
2. Under **Identity**, change to `NetworkService`
3. In SQL Server Management Studio, add a login:
   ```sql
   CREATE LOGIN [NT AUTHORITY\NETWORK SERVICE] FROM WINDOWS;
   USE SportsStats;
   CREATE USER [NT AUTHORITY\NETWORK SERVICE] FOR LOGIN [NT AUTHORITY\NETWORK SERVICE];
   ALTER ROLE db_owner ADD MEMBER [NT AUTHORITY\NETWORK SERVICE];
   ```

Option B: Grant the App Pool identity directly
```sql
CREATE LOGIN [IIS AppPool\SportsStats] FROM WINDOWS;
USE SportsStats;
CREATE USER [IIS AppPool\SportsStats] FOR LOGIN [IIS AppPool\SportsStats];
ALTER ROLE db_owner ADD MEMBER [IIS AppPool\SportsStats];
```

### 3f. Verify web.config

Check that `C:\inetpub\SportsStats\web.config` contains:
```xml
<aspNetCore processPath="dotnet"
            arguments=".\SportsStats.API.dll"
            stdoutLogEnabled="false"
            hostingModel="inprocess" />
```
This file is auto-generated by `dotnet publish`.

### 3g. Browse

Open `http://localhost` (or `http://localhost:8080`) in your browser.

## Republishing after changes

Every time you make changes:
```
# Rebuild React
cd client
npm run build
xcopy /E /Y dist\* ..\SportsStats.API\wwwroot\

# Republish .NET
cd SportsStats.API
dotnet publish -c Release -o C:\inetpub\SportsStats
```
IIS picks up the new files immediately (inprocess hosting restarts automatically).
